Images

img

picture

The Picture element allows forcing the browser to choose a given source, with rigid rules

srcset

srcset "declares" different versions of the image, for different sizes. The browser then decides which is best. This is quite future-proof, because browsers will get better at figuring out what they need

combined srcset and picture

The two techniques above can be combined for maximum flexibility

Image formats

Svg

  • Very light (for vector-like images)
  • Works at any scale
  • Transparency
  • can be changed through css
  • IE8 and older does not support it
  • Does not work well for photos (and is heavier)

jpg

  • Best choice for photographs
  • lossy compression
  • lossy compression

png

  • Best choice for line drawings, images with text, etc...
  • transparency
  • lossless compression
  • lossless compression

inline

  • no additional http requests
  • slower materialization
  • heavier than regular images
  • can't be cached separately

webp

  • lossy and lossless compression can be used
  • lossy compression smaller than jpg
  • lossless compression smaller than png
  • supports animation
  • supports transparency
  • only available in a few browsers
